# MiniService

Мини-веб-служба на ASP.NET Core, демонстрирующая обработку HTTP-запросов через middleware-конвейер.
Сервис хранит данные в памяти процесса и предоставляет простой REST API для работы с элементами предметной области.

##Функциональность

### Сервис реализует три точки доступа:
	•	GET /api/items — получить список элементов
	•	GET /api/items/{id} — получить элемент по идентификатору
	•	POST /api/items — создать новый элемент

Данные хранятся в памяти приложения (без базы данных).

### Пример модели элемента
```
{
  "id": "guid",
  "name": "string",
  "price": 100
}
```

### Валидация

Перед созданием элемента выполняются проверки:
	•	name не может быть пустым
	•	price не может быть отрицательным

При ошибке возвращается единый формат ответа.

Формат ошибок
```
{
  "code": "validation_error",
  "message": "Name must not be empty",
  "requestId": "0HMSF..."
}
```

requestId используется для поиска запроса в логах.

Middleware конвейер

В приложении реализовано три middleware:
	1.	ExceptionMiddleware — перехватывает исключения и формирует единый JSON-ответ.
	2.	TimingMiddleware — измеряет время выполнения запроса.
	3.	Logging — информация о запросе записывается в лог вместе с requestId.

Таким образом можно проследить полный путь обработки запроса.

## Запуск проекта

Требуется установленный .NET SDK.

dotnet restore
dotnet run

После запуска сервис будет доступен по адресу:

```http://localhost:5190```

Примеры запросов

Получить список элементов:

```curl http://localhost:5190/api/items```

Создать элемент:

```
curl -X POST http://localhost:5190/api/items \
-H "Content-Type: application/json" \
-d '{"name":"Test item","price":100}'
```

Получить элемент по id:
```
curl http://localhost:5190/api/items/{id}
```
